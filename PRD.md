# 產品需求文件 (PRD) - TabSync 🌐✨

## 1. 產品概述 (Product Overview)
**TabSync** 是一款基於 Chrome 擴充功能與 Cloudflare Serverless 架構打造的「智能分頁統整助理」。它旨在解決現代知識工作者在瀏覽網頁時產生的「分頁過載 (Tab Overload)」與「資訊碎片化」問題。透過一鍵爬取、AI 總結、與動態翻譯，幫助使用者快速提煉龐大網頁中的核心價值。

## 2. 目標受眾 (Target Audience)
*   **知識工作者 / 研究員**：需要同時查閱大量文獻、文章，並快速提取重點。
*   **跨國商務人士**：需要頻繁閱讀外語資料，並整理成母語報告。
*   **學生 / 開發者**：經常開啟數十個參考資料分頁，需要一個工具來快速歸納或轉為特定格式 (如表格、條列筆記)。

## 3. 痛點分析 (Problem Statement)
*   **資訊焦慮**：開啟過多分頁卻無暇逐一閱讀，最終只能全部關閉，錯失重要資訊。
*   **語言隔閡**：閱讀外文長篇資料耗時費力，依賴第三方翻譯軟體需要反覆複製貼上。
*   **整理成本高**：手動將多個網頁的資訊統整成報告或表格非常耗時。
*   **高昂的 AI API 成本**：市面上的 AI 摘要工具多需付費訂閱，或需要使用者自備 OpenAI API Key，門檻過高。

## 4. 核心功能需求 (Key Features)

### 4.1 智能分頁處理 (Smart Tab Scraping)
*   **自動過濾**：自動排除無法爬取或不需要爬取的系統分頁 (如 `chrome://`、`about:blank`)。
*   **一鍵抓取**：透過注入 Content Script，提取有效分頁的標題 (`<title>`) 與內文 (`<p>`, `<h1>`~`<h3>`)。
*   **大量文檔支援**：單次抓取最高支援 15,000 字元長度，滿足多個長篇網頁的統整需求。

### 4.2 Cloudflare AI 雲端摘要 (AI Summarization)
*   **即時生成**：透過 Cloudflare Workers AI 呼叫 `Llama-3.1-8b-instruct` 模型進行高效能摘要。
*   **自訂指令 (Custom Prompt)**：使用者可輸入客製化指令 (如：「只提取報價資訊並轉為表格」)，作為 AI 處理的最高指導原則。
*   **強制語系輸出**：無條件遵守使用者設定的目標語言進行輸出，解決多語言混合的網頁產生的 AI 語系混亂問題。

### 4.3 零延遲全介面多國語系 (Zero-latency i18n UI)
*   **支援五國語言**：繁體中文、簡體中文、日本語、English、한국어。
*   **即時切換**：透過 `i18n.js` 動態抽換 DOM 元素，切換語言不需重新載入擴充功能。
*   **偏好記憶**：自動儲存使用者的語言選擇至 `chrome.storage.local`。

### 4.4 隱形數據飛輪 (Data Flywheel)
*   **無感記錄**：每次的 AI 請求 (來源網址、爬取內容、AI 生成摘要) 將非同步寫入 Cloudflare D1 (SQL 資料庫)。
*   **未來擴充性**：收集的真實世界優質數據，將作為未來微調 (Fine-tuning) 專屬領域模型 (Domain-specific Model) 的黃金資料集。

## 5. 使用者體驗流程 (User Flow)
1. 使用者在 Chrome 中開啟多個感興趣的網頁。
2. 點擊擴充功能列的 TabSync 圖示，開啟 Popup 面板。
3. (選擇性) 從下拉選單選擇希望輸出的語言 (預設為日語)。
4. (選擇性) 在輸入框填寫自訂的 AI 指令。
5. 點擊「AI 雲端整理所有分頁」。
6. UI 顯示「雲端運算中...」，背景發送資料至 Cloudflare Worker。
7. AI 回傳 Markdown 格式的摘要報告，顯示於下方結果框。
8. 使用者點擊「複製內容」將結果貼至筆記軟體中。

## 6. 非功能性需求 (Non-functional Requirements)
*   **效能 (Performance)**：前端 UI 操作必須在 100ms 內反應 (包含語系切換)。AI 請求應透過流暢的 Loading 動畫安撫使用者等待時間。
*   **架構 (Architecture)**：採用無伺服器 (Serverless) 架構。前端使用原生 HTML/CSS/JS 以確保極致輕量與無外部依賴；後端依賴 Cloudflare 生態系確保全球低延遲與零營運成本。
*   **安全性 (Security)**：
    *   移除前端暴露 API Key 的風險，所有 AI 呼叫皆在後端 Worker 執行。
    *   透過 CORS 標頭限制，防範非預期的跨域請求。

## 7. 未來迭代藍圖 (Future Roadmap)
*   **Phase 2**：加入打字機串流輸出 (Streaming Response) 效果，提升 AI 回覆時的視覺體驗。
*   **Phase 3**：開發 Web Dashboard，讓使用者登入後可以搜尋、回顧過去爬取過的所有摘要紀錄。
*   **Phase 4**：利用 D1 收集的數據，微調專屬的輕量級開源模型，進一步降低推論成本並提升特定任務 (如報價單提取) 的準確率。
